# 注册页交互说明（简单注册）

**版本:** 1.0  
**日期:** 2026年4月  
**关联需求:** [功能详解 — 3.1 用户注册](../../product/features.md#31-用户注册当前内测版简单注册) · [实现状态](../../implementation-status.md)

---

## 一、页面与路由

| 项 | 说明 |
|----|------|
| 路径 | `/register`（公开，无需登录） |
| 布局 | 与登录页同一套**居中卡片**布局（`Auth` 区域），保持视觉与组件一致 |
| 标题 | 如「创建账号」或「注册」 |
| 辅助文案 | 一句话说明需填写邀请码（可选，降低误填率） |

---

## 二、表单字段与顺序

自上而下：

1. **用户名** — 单行文本，`autocomplete="username"`
2. **密码** — 密码输入，`autocomplete="new-password"`，可带显示/隐藏切换
3. **邀请码** — 单行文本；占位符可写「内测邀请码」

不强制「确认密码」字段（当前范围保持三项）；若实现侧增加确认密码，须与产品确认后再改本文档。

---

## 三、校验与反馈

### 3.1 前端校验（提交前）

| 字段 | 规则 |
|------|------|
| 用户名 | 非空；长度与字符集与后端契约一致（缺省可在失焦或提交时提示） |
| 密码 | 非空；满足产品 [密码安全](../../product/features.md#33-安全管理)（≥8 位、大小写+数字） |
| 邀请码 | 非空；**具体有效值由服务端 `auth.invite_code`（或环境变量 `AUTH_INVITE_CODE`）决定**，前端不做硬编码比对；错误文案由接口返回 |

### 3.2 提交中

- 主按钮显示 loading，禁用表单，防止重复提交

### 3.3 提交结果

| 结果 | 交互 |
|------|------|
| 成功 | Toast 或卡片内成功提示 → **跳转 `/login`**；可选：预填用户名（query 或 sessionStorage，勿存密码） |
| 失败（业务错误） | 在表单顶部或对应字段展示服务端 message；常见：用户名已存在、邀请码无效（双保险） |
| 失败（网络） | 通用网络错误提示，保留已填内容（密码可按安全策略清空） |

---

## 四、导航与防呆

| 场景 | 行为 |
|------|------|
| 登录页 | 提供「没有账号？去注册」链到 `/register` |
| 注册页 | 提供「已有账号？去登录」链到 `/login` |
| 已登录用户访问 `/register` | **重定向到 `/`（仪表盘）**，避免重复注册 |

---

## 五、无障碍与主题

- 标签与输入框显式关联（`label` + `htmlFor` / `aria-label`）
- 错误信息关联 `aria-describedby` / `aria-invalid`
- 支持浅色 / 深色主题，与登录页一致

---

## 六、API 契约（供联调）

注册请求体至少包含：`username`、`password`、`invite_code`（字符串）。服务端将 `invite_code`（trim 后）与配置项 **`auth.invite_code`** 比对，完全一致才通过；也可用环境变量 **`AUTH_INVITE_CODE`** 覆盖配置。若配置中邀请码为空字符串，表示**关闭自助注册**。

成功响应后前端按第三节处理跳转；具体状态码与错误体与后端 OpenAPI / 契约文档对齐。

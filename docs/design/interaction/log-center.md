**关联需求:** [请求审计日志（产品）](../../product/audit-log.md) · [导航设计](navigation.md) · [前端架构 — logs 模块](../../architecture/frontend-architecture.md) · [开发 API — 审计](../../development/api.md)

## 一、范围与分期

| 分期 | 内容 | 说明 |
|------|------|------|
| **P0（本期）** | 请求日志列表、请求日志详情、导出 | 与 `GET/POST /api/v1/logs/*` 请求类审计接口对齐 |
| **预留** | 「系统日志」独立子页 | 信息架构见 [设计概述 — 四、信息架构](overview.md)；待产品定义数据源与 API 后再补交互与实现 |

本文 **P0** 默认指「请求审计日志」控制台体验；预留能力在 UI 上可采用 **次级 Tab 占位 + 禁用或「即将推出」** 文案，避免与未就绪后端耦合。

## 二、路由与布局

| 路径 | 页面 | 鉴权 |
|------|------|------|
| `/logs` | 请求日志列表（日志中心首页） | 需登录 |
| `/logs/:requestId` | 单条请求日志详情 | 需登录 |

**布局：** 与控制台主布局一致（顶栏 + 侧栏），见 [导航设计](navigation.md)。  
**面包屑：**

- 列表：`首页 > 日志中心 > 请求日志`
- 详情：`首页 > 日志中心 > 请求日志 > {request_id 短显}`（`request_id` 过长时中间省略，Hover Tooltip 显示完整 ID）

**标题栏右侧主操作（列表页）：**

- `[导出]` — 打开导出弹窗（权限不足时隐藏或禁用并 Tooltip 说明）
- `[刷新]` — 以当前筛选条件重新请求列表（尊重分页与排序）

## 三、信息架构（日志中心内）

```
日志中心
├── [Tab] 请求日志     ← P0
└── [Tab] 系统日志     ← 预留；无 API 时不跳转真实列表，展示空态或说明
```

**Tab 规则：**

- 默认选中「请求日志」。
- 「系统日志」在未就绪时：Tab 可显示但点击后内容为说明卡片（「功能开发中」）或 Tab 整体 `disabled` + Tooltip，二选一由实现与产品拍板，**禁止** 伪造数据表格。

## 四、请求日志 — 列表页

### 4.1 页面结构（桌面）

```
┌─────────────────────────────────────────────────────────────────────────┐
│  筛选区（可折叠「高级筛选」）                                              │
│  时间范围 · 状态 · 模型 · 应用 · 用户/API 密钥/渠道（按角色可见）· 关键词 · [查询] [重置]   │
│  （高级：Finish 原因、token 区间等）                                        │
├─────────────────────────────────────────────────────────────────────────┤
│  表格（可排序列 + 行操作）                              [列设置 ▼] 可选    │
├─────────────────────────────────────────────────────────────────────────┤
│  分页：每页条数 · 共 N 条 · 上一页/下一页 · 页码                            │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 筛选条件

| 控件 | 行为 | 说明 |
|------|------|------|
| **时间范围** | 日期时间范围选择器；提供快捷项：今天 / 近 7 天 / 近 30 天 | 必选其一有效区间；默认「近 7 天」 |
| **HTTP 状态** | 多选或分段控件：全部 / 2xx / 4xx / 5xx | 与产品 [audit-log](../../product/audit-log.md)「按状态码筛选」一致 |
| **模型** | 可搜索下拉，支持多选 | 选项来自列表聚合或独立枚举接口（以后端为准） |
| **应用（App）** | 可搜索下拉，支持单选或多选（与接口一致） | 标识调用方业务应用（如 `app_id`、应用名称）；选项来自枚举或列表聚合接口；**无应用维度时整项不展示** |
| **用户** | 可搜索下拉 | **超级管理员 / 普通管理员** 可见；普通用户不展示（固定为本人数据） |
| **API 密钥** | 可搜索下拉 | 对应 `token_id`，按权限裁剪 |
| **渠道** | 可搜索下拉 | 同上 |
| **关键词** | 单行搜索，防抖 300ms | 匹配 `request_id`、错误信息等后端约定字段；占位符说明可搜索范围 |
| **查询** | 提交筛选，回到第 1 页 | 触发列表请求 |
| **重置** | 恢复默认筛选 | 时间回到默认快捷项，清空其他条件 |

**高级筛选：**

- 折叠面板内放置低频维度（如 `request_type`、最小/最大 latency、**prompt / completion token 区间**（分项或合计，以后端为准）、**Finish 原因 (`finish_reason`)** 多选等），以 **后端实际支持的 query 参数** 为准扩展；未支持的不展示。

### 4.3 URL 与状态同步

- 筛选条件、页码、每页条数、排序字段与方向 **写入 URL Query**（如 `?start=...&end=...&app=...&page=1&sort=created_at&order=desc`），参数字段名 **与后端 OpenAPI 一致**（示例中的 `app` 仅为占位），便于分享与刷新不丢状态。
- 从详情页返回列表时：优先 **恢复离开前的 URL**（浏览器历史），与上述约定一致。

### 4.4 表格列（默认建议）

| 列 | 排序 | 说明 |
|----|:----:|------|
| 时间 (`created_at`) | ✅ 默认倒序 | 格式：本地化日期时间 |
| `request_id` | 可选 | 等宽字体；可复制 |
| 模型 | — | |
| 应用（App） | 可选 | 展示应用标识或名称（字段名以后端列表项为准，如 `app_id` + 展示名）；无值显示 `—` |
| HTTP 状态 | ✅ | Badge：2xx 绿 / 4xx 橙 / 5xx 红（色弱模式仅靠图标+文字区分） |
| 输入 Tokens（prompt） | ✅ | `prompt_tokens`；无值 `—` |
| 输出 Tokens（completion） | ✅ | `completion_tokens`；无值 `—` |
| 合计 Tokens（可选） | 可选 | `total_tokens`；可与上两列并存作校验展示，或 **列设置** 中默认关闭以省宽 |
| **Finish 原因** | 可选 | 对应响应中的 `finish_reason`（与 [API 文档 — Chat 响应](../../product/api.md) 语义一致，如 `stop`、`length`、`content_filter`、`tool_calls` 等）。**展示：** 单元格显示简短文案（可做中英文映射）；**完整枚举值** 用 Tooltip。不适用场景（非 chat/completion、请求失败无有效 completion）显示 `—`。多候选时取值规则 **由后端定义**（如取首条 `choices[0]` 或主 completion），前端与 OpenAPI 对齐 |
| 耗时 (`latency_ms`) | ✅ | 毫秒，可读格式（如 `1.2s`） |
| 成本 (`cost`) | ✅ | 货币格式，权限不足时可隐藏列 |
| 操作 | — | `查看详情` |

**列设置（可选 P1）：** 用户可显隐列（含「合计 Tokens」「Finish 原因」）；偏好存 `localStorage`。

**窄屏（`< md`）表格：** 可将 prompt / completion 合并为 **单列**「Tokens」，单元格内两行文案：`输入 · {prompt}` / `输出 · {completion}`，或使用紧凑格式 `{prompt} / {completion}` + 列头 Tooltip 说明前后含义。

### 4.5 行交互

- **单击行** 或 **点击「查看详情」**：跳转 `/logs/{requestId}`。
- **复制 request_id**：行内图标按钮，成功 Toast「已复制」。

### 4.6 分页

- 每页条数：10 / 20 / 50 / 100（上限以后端 `page_size` 为准，产品文档示例为 100）。
- 显示总条数；若后端返回「超过单次上限」，文案提示「仅展示当前页结果，请缩小时间范围或导出」。

### 4.7 空态、加载、错误

| 状态 | UI |
|------|-----|
| 首次加载 | 表格骨架屏（Skeleton） |
| 无数据 | 插图或图标 + 「所选条件下暂无请求日志」+ 引导调整筛选或导出历史 |
| 接口错误 | 表格区域错误占位 + `[重试]`；403 说明权限；429 说明限流稍后重试 |
| 部分字段缺失 | 单元格显示 `—`，不闪错 |

## 五、请求日志 — 详情页

### 5.1 页面结构

```
┌────────────────────────────────────────────────────────────┐
│  [← 返回列表]  request_id（可复制）  [在列表中打开] 可选    │
├────────────────────────────────────────────────────────────┤
│  摘要卡片：时间 · 状态码 · 模型 · 应用（App）· 用户/API 密钥/渠道 · 输入 Tokens · 输出 Tokens · 合计（可选）· Finish 原因 · 耗时 · 成本 │
├────────────────────────────────────────────────────────────┤
│  Tab：[请求体] [响应体] [用量详情] [成本详情]（无数据 Tab 隐藏或禁用）   │
│  内容区：语法高亮 JSON · 支持折叠 · 搜索（可选）· 复制 / 下载片段       │
└────────────────────────────────────────────────────────────┘
```

### 5.2 摘要区

- 字段只读；敏感字段（如完整 API Key）**永不展示**，与 [API 文档 — 日志脱敏](../../product/api.md) 原则一致。
- **用量：** 与列表一致，**区分** `prompt_tokens`（输入）、`completion_tokens`（输出）；`total_tokens` 可作为「合计」单独展示，或与两者之和一致时用于对照（不一致时仍以分项为准并可在「用量详情」Tab 解释）。
- **Finish 原因：** 与列表列一致，展示 `finish_reason`；多候选、流式或未结束时与后端约定一致（无则 `—`）。完整技术枚举可在 Tooltip 或「响应体」Tab 核对。
- 长文本（`error_message`）完整显示在摘要下方或独立 Alert 区块。

### 5.3 JSON 展示

- **默认折叠** 深层节点，首屏展示顶层键。
- **复制**：复制当前 Tab 下格式化 JSON 或原始字符串（实现二选一，需在组件内统一）。
- **超大响应**：虚拟滚动或「仅展示前 N KB + 下载全文」提示，避免卡死主线程（具体阈值与后端分页/截断策略对齐）。

### 5.4 加载与错误

- 加载：摘要区 Skeleton + 内容区 Spinner。
- `404`：「未找到该请求或无权访问」+ 返回列表。
- `403`：无权查看他人日志时的说明 + 返回列表。

## 六、导出

### 6.1 触发

- 列表页 `[导出]` 打开 **模态对话框**（`role="dialog"`, `aria-modal="true"`），见 [设计概述 — 无障碍](overview.md)。

### 6.2 导出弹窗字段

| 字段 | 说明 |
|------|------|
| 时间范围 | 默认与列表当前筛选一致，可改 |
| 格式 | 单选：`JSON` / `CSV` |
| 字段 | 多选勾选，默认一组常用字段（含 **`prompt_tokens`、`completion_tokens`、`finish_reason`**，可选 `total_tokens`；含 **应用** 相关列若后端支持）；全选 / 恢复默认 |
| 预估 | 若接口返回预估条数或体积，展示提示；否则显示「导出任务已创建」类中性文案 |

**主按钮：**「创建导出」；提交中 loading，防重复点击。

### 6.3 导出任务状态

1. 创建成功后展示 `export_id`（可折叠）与状态：`processing` / `completed` / `failed`。
2. **轮询** 间隔建议 2s，超时（如 5 分钟）提示联系管理员或缩小范围。
3. `completed`：展示 `[下载]`，新开或直链下载 `GET .../export/{id}/download`（与 [audit-log API 示例](../../product/audit-log.md) 一致）。
4. `failed`：错误原因 + `[关闭]` / `[重试]`（重试 = 重新打开弹窗并保留条件）。

### 6.4 权限

- 无导出权限：不展示导出按钮或禁用并说明（与 [audit-log — 权限控制](../../product/audit-log.md) 一致）。

## 七、响应式与移动端

与 [设计概述 — 六、响应式设计](overview.md) 一致：

| 断点 | 行为 |
|------|------|
| `< md` | 筛选区改为纵向堆叠 + 底部固定「确定筛选」操作条（可选）；表格横向滚动，优先保留时间、状态、模型、应用（App）、**Tokens（合并列或 P/C 两列缩略）**、操作列；**Finish 原因** 可默认隐藏，由用户通过列设置或横滑查看 |
| 详情页 | 摘要卡片单列堆叠；JSON 区全宽，字号略小仍保持等宽 |

日志中心 **未** 列入移动端底部 Tab 主入口时，从抽屉「日志中心」进入，见 [导航设计 — 三、移动端导航](navigation.md)。

## 八、权限与数据范围

| 角色 | 列表与详情 | 导出 |
|------|------------|------|
| 超级管理员 | 全量（含用户/API 密钥/渠道/应用筛选） | 允许 |
| 普通管理员 | 全量或产品约定范围 | 允许 |
| 普通用户 | 仅本人相关记录；隐藏「用户」筛选或固定只读展示自己；**应用**筛选按租户策略显示（通常保留，仅可选本人数据范围内的应用） | 仅本人范围 |

前端以 **401/403 与业务错误码** 为准展示文案，不与后端硬编码权限表重复实现业务规则。

## 九、无障碍与国际化

- 表格：`<table>` 语义或 `role="table"`，列头 `scope="col"`。
- 筛选控件：`label` 与 `id` 关联；错误用 `aria-invalid` + `aria-describedby`。
- 状态 Badge：不仅依赖颜色，附带文字或图标。
- **Finish 原因** 以文本为主（必要时辅以浅色 Tag），勿仅用颜色区分枚举。
- 时间：使用用户时区展示，ISO 8601 存传。
- 文案抽 key，便于后续英文等语言（与全局 i18n 策略一致）。

## 十一、验收要点（交互）

1. 列表：筛选、分页、排序、URL 同步、空/错/载状态完整；**应用（App）** 列与筛选与后端契约一致，无该能力时整列/筛选项隐藏；**输入/输出 Tokens 分列展示**（窄屏按第四节合并规则处理）；**Finish 原因** 列与列表接口字段一致，无该能力时整列/高级筛选项隐藏。
2. 详情：摘要清晰、JSON 可读、大 body 不拖死页面、无敏感密钥泄露。
3. 导出：创建 → 轮询 → 下载闭环；失败可理解、可重试。
4. 权限：各角色可见控件与数据范围符合上文；越权有明确提示。
5. 键盘可达：筛选 → 表格 → 详情 → 返回，焦点顺序合理。

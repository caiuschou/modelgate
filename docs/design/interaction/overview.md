# ModelGate 交互设计概述

**版本:** 1.0
**日期:** 2026年3月
**文档类型:** 交互设计规格说明书

---

## 一、设计目标

- **高效操作** — 核心任务 3 步内完成
- **信息清晰** — 关键数据一目了然
- **降低门槛** — 新用户 5 分钟上手
- **专业可信** — 企业级视觉品质

---

## 二、设计原则

| 原则 | 说明 |
|------|------|
| 一致性 | 相同功能采用一致的交互模式 |
| 反馈即时 | 操作后 200ms 内给予反馈 |
| 容错性 | 危险操作需二次确认，支持撤销 |
| 渐进披露 | 高级功能折叠，不干扰主流程 |

---

## 三、用户角色

| 角色 | 权限 | 核心任务 |
|------|------|---------|
| 超级管理员 | 全部 | 渠道配置、用户管理、系统设置 |
| 普通管理员 | 部分 | 令牌管理、查看统计 |
| 普通用户 | 基础 | 查看配额、使用 API |

## 五、用户流程

### 5.1 首次使用流程

```
注册账号 → 验证邮箱 → 登录系统 → 添加渠道 → 测试连通性 → 创建令牌 → 开始调用
```

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  注册账号  │───→│  验证邮箱  │───→│  登录系统  │───→│  添加渠道  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                    │
┌──────────┐    ┌──────────┐    ┌──────────┐        │
│  开始调用  │←───│  复制Key  │←───│  创建令牌  │←───────┘
└──────────┘    └──────────┘    └──────────┘
```

### 5.2 日常使用流程

```
登录 → 查看仪表盘 → 检查用量/告警 → 调整配置（如需）→ 继续使用
```

### 5.3 故障处理流程

```
收到告警 → 查看日志 → 定位问题 → 处理（禁用/切换/联系供应商）→ 验证恢复
```

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  收到告警  │───→│  查看日志  │───→│  定位问题  │
└──────────┘    └──────────┘    └──────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
              ┌──────────┐    ┌──────────┐    ┌──────────┐
              │ 禁用渠道  │    │ 切换备用  │    │ 联系供应商 │
              └──────────┘    └──────────┘    └──────────┘
                    │               │               │
                    └───────────────┼───────────────┘
                                    ↓
                              ┌──────────┐
                              │  验证恢复  │
                              └──────────┘
```

## 八、无障碍设计

### 8.1 基本要求

- 所有图片提供 alt 属性
- 表单控件关联 label
- 链接具有描述性文字
- 颜色对比度 ≥ 4.5:1
- 支持键盘导航

### 8.2 ARIA 标签

| 组件 | ARIA 属性 |
|------|----------|
| 按钮 | role="button" |
| 弹窗 | role="dialog", aria-modal="true" |
| Tab | role="tablist", role="tab" |
| 表格 | role="table" |
| 加载 | role="status", aria-live="polite" |

## 十、异常流程处理

### 10.1 网络异常

**全局网络断开：**
```
┌────────────────────────────────────────────────────────────┐
│ ⚠️ 网络连接已断开，正在尝试重连...              [刷新页面] │
└────────────────────────────────────────────────────────────┘
```

- 位置：页面顶部固定条
- 背景：`#FFFBE6`
- 自动重连：每 5 秒尝试一次
- 重连成功：显示 Toast "网络已恢复"

### 10.2 会话过期

**弹窗提示：**
```
┌────────────────────────────────────┐
│  会话已过期                   [×]  │
├────────────────────────────────────┤
│                                    │
│    🔐 您的登录已过期                │
│    请重新登录以继续使用             │
│                                    │
│                  [取消]  [重新登录] │
└────────────────────────────────────┘
```

- 触发：API 返回 401
- 行为：点击重新登录跳转登录页
- 记忆：保存当前页面 URL，登录后跳回

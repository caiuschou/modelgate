# ModelGate 前端开发计划

**版本:** 1.2  
**制定日期:** 2026年4月1日  
**更新说明:** 按 Feature 拆分任务并增加状态字段；注册能力提前至鉴权阶段  
**项目周期:** 8周  
**适用范围:** ModelGate 管理控制台（React + Vite + shadcn/ui + TailwindCSS）

---

## 一、计划概述

### 1.1 目标

基于现有 `docs/architecture/frontend-architecture.md`，在 8 周内交付可用于内测的前端管理控制台，覆盖以下核心能力：

- 注册、登录与基础权限控制
- 仪表盘数据展示
- 日志中心（查询、详情、导出）
- 令牌管理（创建、查看、复制、吊销）
- 用户管理（基础增查）
- 渠道管理（基础增改查、测试）

### 1.2 范围边界

**本期包含（P0 + 部分 P1）**

- Web 端管理控制台（桌面优先，兼容平板/移动）
- 前后端 API 集成
- 基础可观测性（前端错误追踪、关键埋点）
- 单元测试 + 关键流程 E2E

**本期不包含（后续迭代）**

- 多语言完整翻译（仅保留 i18n 框架能力）
- 高级 BI 看板定制
- 复杂 RBAC 权限策略编辑器
- 移动端 App

## 二、里程碑与阶段

| 阶段 | 周期 | 目标 | 里程碑产出 |
|------|------|------|-----------|
| Phase 1 | Week 1-2 | 工程骨架与基础设施 | 可运行前端骨架、路由、鉴权（含注册）、CI 检查 |

| Phase 3 | Week 5-6 | 核心业务页面第二批 | 用户管理、渠道管理、统计分析初版 |

## 五、前后端联调计划

### 5.1 已有可用 API（当前可联调）

- `GET /healthz`
- `POST /users`
- `POST /users/{username}/keys`
- `GET /api/v1/logs/request`
- `GET /api/v1/logs/request/{request_id}`
- `POST /api/v1/logs/export`
- `GET /api/v1/logs/export/{export_id}`
- `GET /api/v1/logs/export/{export_id}/download`

### 5.2 待后端补齐 API（前端关键阻塞）

- 认证：`/api/v1/auth/register`（body：`username`、`password`、`invite_code`；`invite_code` 须与配置 `auth.invite_code` 或环境变量 `AUTH_INVITE_CODE` 一致）、`/api/v1/auth/login`、`/api/v1/auth/me`、`/api/v1/auth/logout`
- 用户列表/更新/删除
- 令牌列表/更新/删除
- 渠道列表/创建/更新/删除/测试
- 统计分析接口（overview/usage/models）

### 5.3 联调策略

- 先用 Mock（MSW）并行开发页面
- 接口可用后切换真实 API
- 每周固定 2 次联调窗口（建议周二、周五）
- 对齐接口契约：字段命名、分页格式、错误码

## 七、质量与发布标准

### 7.1 质量门禁

- lint 无 error
- TypeScript 无 error（`tsc --noEmit`）
- 单测通过率 100%
- E2E 关键流程通过
- 关键页面无阻塞级可用性问题

### 7.2 性能指标（建议目标）

- 首屏 LCP < 2.5s（预发布环境）
- 首屏 JS gzip < 120KB（核心路由）
- 交互响应（点击到反馈）< 200ms

### 7.3 发布节奏

- 每周一次小版本发布（预发布）
- 双周一次稳定版本发布（生产）
- 紧急修复走 hotfix 分支

## 场景一：企业内部网关

### 场景描述

企业多个团队需要使用不同的大模型，各自管理 API Key 导致混乱，成本无法控制。

### 解决方案

部署 ModelGate 作为统一网关，各团队通过分配的 **API 密钥** 调用，管理员统一管控渠道、配额、权限。

### 核心价值

- **统一管理** — 避免 Key 泄露
- **成本透明** — 按团队/项目分摊
- **权限隔离** — 保障数据安全

### 实施流程

```
1. 部署 ModelGate 服务器
2. 配置企业常用的模型渠道
3. 为各团队创建用户账号
4. 按项目分配 API 密钥和配额
5. 团队使用 API 密钥调用 API
6. 管理员监控用量和成本
```

### 典型配置

**渠道配置：**
- OpenAI GPT-4（主力模型）
- Claude 3.5（备用模型）
- DeepSeek（成本优化）
- 通义千问（国内合规）

**API 密钥与用量配额（示例）：**
- 研发团队 API 密钥：100 万 **用量** Token/月
- 产品团队 API 密钥：50 万 **用量** Token/月
- 测试团队 API 密钥：10 万 **用量** Token/月

## 场景二：API 二次分发

### 场景描述

AI 服务商希望整合多家模型能力，对外提供统一的 API 服务，按量收费。

### 解决方案

ModelGate 聚合多供应商模型，通过 **API 密钥** 分发给终端用户，内置计费与用量统计。

### 核心价值

- **一次接入** — 多模型可用
- **内置计费** — 降低开发成本
- **高可用架构** — 保障服务稳定

### 商业模式

**收费模式：**
- 按量计费：¥0.15/1K Token（GPT-4）
- 套餐订阅：¥99/月，含50万Token
- 企业定制：按需报价

**盈利方式：**
- 批量采购获得折扣
- 差价收费
- 增值服务（技术支持、定制开发）

### 系统架构

```
用户 → ModelGate → 多个供应商
             ↓
         计费统计
             ↓
         用户后台
```

## 场景三：成本控制与优化

### 场景描述

企业使用多个模型，成本持续增长，但缺乏有效的监控与控制手段。

### 解决方案

ModelGate 提供精细化用量统计，设置预算上限，超出自动限流或告警。

### 核心价值

- **用量透明** — 成本可控
- **预算预警** — 防止超支
- **数据驱动** — 优化模型选择

### 成本优化策略

**1. 模型分层使用：**

| 任务类型 | 使用模型 | 成本 |
|---------|---------|------|
| 简单问答 | GPT-3.5 / DeepSeek | ¥0.01/1K Token |
| 复杂推理 | GPT-4 / Claude 3.5 | ¥0.15/1K Token |
| 代码生成 | Claude 3.5 Sonnet | ¥0.03/1K Token |
| 翻译 | 通义千问 | ¥0.008/1K Token |

**2. 配额管理：**
- 部门配额：按月分配
- 项目配额：按项目周期分配
- 个人配额：按角色分配

**3. 智能调度：**
- 低优先级任务自动切换到低成本模型
- 高峰期使用备用渠道
- 缓存常见问题

### 成本监控

**实时监控指标：**
- 今日消费：¥2,580
- 本月累计：¥45,680

**预警机制：**

## 场景四：多模型 A/B 测试

### 场景描述

开发团队希望快速对比不同模型的效果与成本，选择最优方案。

### 解决方案

通过 ModelGate 配置多个渠道，切换模型只需修改请求参数，无需修改代码。

### 核心价值

- **快速切换** — 降低试错成本
- **统一评估** — 数据驱动决策
- **灰度发布** — 降低风险

### A/B 测试流程

**1. 配置测试渠道：**

```yaml
渠道A: GPT-4 Turbo
  优先级: 5
  权重: 50%
  标签: test-group-a

渠道B: Claude 3.5 Sonnet
  优先级: 5
  权重: 50%
  标签: test-group-b
```

**2. 用户分组：**
- 50% 用户分配到组 A
- 50% 用户分配到组 B

**3. 数据收集：**
- 响应质量评分
- 响应时间
- Token 使用量
- 成本

**4. 结果分析：**

| 指标 | GPT-4 | Claude 3.5 | 差异 |
|------|-------|------------|------|
| 质量评分 | 4.5 | 4.7 | +4.4% |
| 响应时间 | 2.3s | 1.8s | -21.7% |
| 成本 | ¥0.15 | ¥0.03 | -80% |

**5. 决策：**
- 全量切换到 Claude 3.5
- 月节省成本：¥12,000

## 场景五：私有化部署

### 场景描述

金融、医疗等行业对数据安全要求高，需要私有化部署 AI 网关。

### 解决方案

ModelGate 支持本地部署，数据不出内网，完全自主可控。

### 核心价值

- **数据安全** — 满足合规要求
- **自主可控** — 灵活定制
- **高性能** — 低延迟

### 部署方案

**单机部署：**
- 适用于：小型团队、POC 验证
- 配置：4核8G，SSD 100G
- 成本：¥500/月（云服务器）

**高可用部署：**
- 适用于：生产环境
- 配置：3节点集群，负载均衡
- 成本：¥2,000/月

**混合部署：**
- 适用于：多地办公
- 配置：中心节点 + 边缘节点
- 成本：¥5,000/月

### 安全合规

**数据安全：**
- 所有数据在内网传输
- 敏感数据加密存储
- 操作日志完整记录

**访问控制：**
- VPN 接入
- 双因素认证
- IP 白名单

**审计合规：**
- 保留 180 天日志
- 不可删除审计记录
- 支持导出合规报告

### 典型客户

**金融机构：**
- 银行：智能客服、风险控制
- 证券：研报生成、投资建议
- 保险：理赔审核、保单分析

**医疗机构：**
- 医院：病历生成、辅助诊断
- 药企：药物研发、文献分析

**政府机构：**
- 政务：智能问答、文档处理
- 公安：案情分析、预警

## 场景六：开发者工具集成

### 场景描述

开发者希望在自己的 IDE、CLI 工具中直接使用 AI 能力。

### 解决方案

通过 ModelGate 统一 API，轻松集成到各种开发者工具。

### 集成方式

**1. IDE 插件：**
- VS Code 插件
- JetBrains 插件
- 代码补全
- 代码解释

**2. CLI 工具：**
- 命令行调用
- 脚本自动化
- CI/CD 集成

**3. API 集成：**
```bash
curl https://api.modelgate.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### 开发者体验

**快速开始：**
```bash
# 1. 安装 CLI
npm install -g modelgate-cli

# 2. 配置 Token
modelgate config set_token sk-xxx

# 3. 开始使用
modelgate chat "帮我写一个快速排序"
```

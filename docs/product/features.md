# ModelGate 功能详解

**版本:** 1.1  
**更新日期:** 2026年4月

---

## 一、渠道管理功能

### 1.1 新增渠道

**验收标准：**
- 必填字段校验通过后可保存
- 保存成功后列表可见新渠道
- 支持设置优先级和权重
- 保存前可测试连通性

**用户故事：**
```
作为 系统管理员
我想要 添加新的模型供应商渠道
以便于 扩展可用的模型能力
```

### 1.2 批量导入渠道

**验收标准：**
- 支持 JSON 和 CSV 格式
- 导入前预览解析结果
- 导入失败显示具体错误行
- 导入成功后自动测试连通性

### 1.3 渠道健康监控

**验收标准：**
- 仪表盘显示各渠道状态
- 故障时自动发送告警通知
- 可查看历史故障记录
- 支持手动触发健康检查

**状态定义：**
- 🟢 正常：服务正常响应
- 🟡 慢响应：响应时间 > 5s
- 🔴 故障：服务不可用
- ⚪ 禁用：已手动禁用

### 1.4 渠道灰度发布

**验收标准：**
- 可设置渠道可见用户组
- 未开放用户不可见该渠道
- 支持按比例灰度
- 可一键切换为全量

### 1.5 调度规则

**调度优先级：**
1. 匹配模型名称
2. 渠道优先级（数字越大越优先）
3. 渠道权重（同优先级时按权重随机）
4. 渠道状态（故障渠道不参与调度）

**故障判定标准：**

| 条件 | 判定结果 |
|------|---------|
| 连续3次请求超时 | 暂时不可用（5分钟后重试） |
| 连续5次请求失败 | 自动禁用（需手动启用） |
| 响应时间>10秒 | 标记为慢响应 |
| HTTP 5xx 错误 | 计入失败次数 |
| HTTP 4xx 错误 | 不计入失败（客户端错误） |

## 二、令牌管理功能

### 2.1 创建令牌

**验收标准：**
- 3步内完成创建：命名→配额→生成
- 生成后显示完整 Key 仅一次
- 支持一键复制
- 可选设置过期时间

**用户故事：**
```
作为 开发者
我想要 快速生成 API 访问令牌
以便于 在我的应用中调用 API
```

### 2.2 配额管理

**配额类型：**
- 无限制
- 按 Token 数量
- 按金额

**配额规则：**
```
扣减额度 = 输入Token × 输入倍率 + 输出Token × 输出倍率
```

**默认倍率：**

| 模型类型 | 输入倍率 | 输出倍率 |
|---------|---------|---------|
| GPT-4 系列 | 1.0 | 2.0 |
| GPT-3.5 系列 | 1.0 | 1.0 |
| Claude 系列 | 1.0 | 3.0 |
| 国产模型 | 1.0 | 1.0 |
| Embedding | 1.0 | 0 |

**用量数据结构：**

| 字段 | 用途 | 说明 |
|------|------|------|
| prompt_tokens | 配额扣减 | 输入Token数 |
| completion_tokens | 配额扣减 | 输出Token数 |
| total_tokens | 统计/预警 | 总Token数 |
| cost | 配额扣减 | 消费金额（元） |
| cached_tokens | 缓存优化 | 缓存命中不收费 |

**配额预警：**
- 使用量达80%显示黄色预警
- 使用量达95%显示红色告警

### 2.3 权限控制

**接口权限矩阵：**

| 接口 | 普通令牌 | 管理员令牌 |
|------|:-------:|:---------:|
| /v1/chat/completions | ✅ | ✅ |
| /v1/completions | ✅ | ✅ |
| /v1/embeddings | ✅ | ✅ |
| /v1/images/generations | 可配置 | ✅ |
| /v1/audio/* | 可配置 | ✅ |
| /v1/models | ✅ | ✅ |
| 管理接口 | ❌ | ✅ |

**IP白名单规则：**
- 未配置：不限制IP
- 已配置：仅允许列表内IP访问
- 支持格式：`192.168.1.1`、`192.168.1.*`、`192.168.1.0/24`

**域名限制规则：**
- 仅对前端调用生效
- 检查请求头 `Origin` 或 `Referer`
- 支持通配符：`*.example.com`

### 2.4 用量分析

**分析维度：**
- 显示请求数、Token数、消费额
- 可按时间范围筛选
- 可按模型分组统计
- 支持导出数据

**统计指标：**

| 指标 | 数据来源 | 用途 |
|------|---------|------|
| 请求数 | request_logs | 调用量统计 |
| 输入Token | prompt_tokens | 输入用量分析 |
| 输出Token | completion_tokens | 输出用量分析 |
| 总Token | total_tokens | 总量统计 |
| 成本 | cost | 成本核算 |
| 缓存命中率 | cached_tokens / prompt_tokens | 缓存优化 |
| 平均延迟 | latency_ms | 性能分析 |

## 三、用户管理功能

### 3.1 用户注册（当前内测版：简单注册）

**范围说明：** 当前阶段采用**极简自助注册**，不启用邮箱验证、第三方注册与验证邮件；与下文「后续能力」区分。

**表单字段：**

| 字段 | 说明 | 规则 |
|------|------|------|
| 用户名 | 登录标识 | 与后端约定一致（如长度、字符集）；不可与已有用户重复 |
| 密码 | 登录凭证 | 与 [3.3 密码安全](#33-安全管理) 一致（最少 8 位，含大小写与数字） |
| 邀请码 | 内测门禁 | **须与服务端配置 `auth.invite_code`（或环境变量 `AUTH_INVITE_CODE`）完全一致**（trim 后比对，区分大小写）；**仅以服务端校验为准** |

**验收标准：**
- 三项均合法且邀请码正确时，注册成功并创建用户
- 邀请码错误、用户名已存在、密码不合规时，返回明确错误提示（不泄露用户是否存在时可统一文案，由安全策略决定）
- 注册成功后引导用户前往登录页（或按实现自动登录，需与前端交互文档一致）
- 不包含：邮箱验证、第三方 OAuth、注册邮件

**用户故事：**
```
作为 新用户
我想要 用用户名、密码和邀请码完成注册
以便于 在受控内测范围内开通账号
```

**后续能力（未纳入当前验收）：** 邮箱注册、第三方登录（GitHub/微信）、验证邮件、首次登录引导 — 见路线图与后续迭代。

### 3.2 多租户管理

**验收标准：**
- 可邀请/移除成员
- 可设置成员角色
- 可查看成员使用情况
- 支持组织级配额管理

**角色权限：**

| 角色 | 权限 | 核心任务 |
|------|------|---------|
| 超级管理员 | 全部 | 渠道配置、用户管理、系统设置 |
| 普通管理员 | 部分 | 令牌管理、查看统计 |
| 普通用户 | 基础 | 查看配额、使用 API |

### 3.3 安全管理

**登录保护：**
- 支持用户名/邮箱登录
- 连续5次失败锁定15分钟
- 异地登录提醒
- 支持强制登出其他设备

**密码安全：**
- 密码加密存储
- 最少8位，含大小写+数字
- 不支持明文找回

## 四、统计分析功能

### 4.1 成本分析

**验收标准：**
- 按日/周/月汇总成本
- 按部门/项目/用户分摊
- 支持同比/环比分析
- 可导出财务报表

**用户故事：**
```
作为 财务人员
我想要 查看详细的成本报表
以便于 进行成本核算和预算编制
```

### 4.2 用量预警

**预警配置：**
- 可设置预警阈值（如80%、90%）
- 支持邮件/短信/站内信通知
- 预警可设置静默期
- 支持多人接收预警

### 4.3 实时统计

**数据指标：**
- 实时请求量
- Token 使用量
- 成本消耗
- 平均响应时间

**数据来源：** request_logs 表实时聚合

**统计更新延迟：** < 5分钟
- 成本消耗
- 平均响应时间

**数据延迟：** < 5分钟

## 五、API 请求处理

### 5.1 请求认证

**认证方式：**
- Bearer Token（HTTP Header）
- API Key（Query Parameter）

**错误码：**

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 缺少令牌 | 401 | Missing API key | 检查请求头是否包含 Authorization |
| 令牌无效 | 401 | Invalid API key | 检查令牌是否正确 |
| 令牌已过期 | 401 | Token expired | 联系管理员续期或创建新令牌 |
| 令牌已吊销 | 401 | Token revoked | 创建新令牌 |
| IP 不在白名单 | 403 | IP not allowed | 联系管理员添加 IP |

### 5.2 请求路由

**路由逻辑：**
1. 根据模型名称匹配渠道
2. 按优先级和权重选择渠道
3. 转发请求到上游
4. 格式化响应返回

**错误处理：**

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 模型不存在 | 404 | Model not found | 检查模型名称是否正确 |
| 无可用渠道 | 503 | No available channel | 联系管理员添加渠道 |
| 上游超时 | 504 | Upstream timeout | 稍后重试 |
| 请求过于频繁 | 429 | Rate limit exceeded | 等待 retry_after 秒后重试 |

### 5.3 流式响应

**支持方式：**
- SSE（Server-Sent Events）
- 流式中断自动重连

**响应时间：**
- 流式首字延迟：P95 < 500ms

## 六、并发与限流

### 6.1 并发限制

| 层级 | 默认限制 | 说明 |
|------|---------|------|
| 系统级 | 1000 QPS | 整体请求上限 |
| 用户级 | 100 QPS | 单用户请求上限 |
| 令牌级 | 可配置 | 单令牌请求上限 |
| 渠道级 | 可配置 | 单渠道请求上限 |

### 6.2 限流策略

- 超限返回 HTTP 429
- 响应头包含 `X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`
- 响应体包含 `retry_after` 字段（秒）

## 七、计费规则

### 7.1 计费模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 预付费 | 先充值后使用，余额不足停服 | 个人用户 |
| 后付费 | 月底结算，超额部分按量计费 | 企业用户 |
| 套餐制 | 固定月费，含定量额度，超额另计 | SaaS 服务 |

### 7.2 充值规则

- 最低充值：10元
- 充值到账：实时
- 发票：支持申请电子发票

### 7.3 退款规则

- 未使用余额可申请退款
- 退款收取5%手续费
- 退款周期：3-5个工作日

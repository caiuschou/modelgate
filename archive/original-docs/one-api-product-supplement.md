## 一、验收标准清单

### 1.1 渠道管理

| 功能点 | 验收标准 | 优先级 |
|-------|---------|--------|
| 新增渠道 | 必填字段校验通过后可保存；保存成功后列表可见新渠道 | P0 |
| 编辑渠道 | 修改内容实时保存；未修改时保存按钮禁用 | P0 |
| 删除渠道 | 弹出二次确认；删除后关联令牌不可用该渠道 | P0 |
| 批量操作 | 支持批量启用/禁用/删除；操作后显示成功数量 | P1 |
| 测试连通性 | 点击后3秒内返回结果；成功显示✅，失败显示原因 | P0 |
| 渠道状态 | 正常🟢/慢响应🟡/故障🔴；状态每60秒刷新 | P0 |
| 模型映射 | 支持自定义模型名称映射；映射后请求使用映射值 | P1 |
| 优先级调度 | 多渠道时按优先级选择；同优先级按权重分配 | P0 |
| 故障转移 | 主渠道故障自动切换备用渠道；切换时间<1秒 | P0 |
| 自动禁用 | 连续失败5次自动禁用；禁用后发送通知 | P1 |

### 1.2 令牌管理

| 功能点 | 验收标准 | 优先级 |
|-------|---------|--------|
| 生成令牌 | 生成后显示完整Key仅一次；支持一键复制 | P0 |
| 令牌配额 | 可设置Token数量/金额上限；超限后请求返回402 | P0 |

| IP白名单 | 配置后非白名单IP请求返回403；支持通配符 | P1 |
| 模型限制 | 限制后请求非允许模型返回403 | P1 |
| 吊销令牌 | 吊销后立即生效；正在进行的请求正常完成 | P0 |
| 令牌详情 | 显示创建时间、使用量、最后调用时间 | P0 |
| 用量统计 | 按令牌维度查看请求数、Token数、消费额 | P1 |

### 1.3 用户管理

| 功能点 | 验收标准 | 优先级 |
|-------|---------|--------|
| 用户注册 | 邮箱格式校验；密码强度校验；注册后发送验证邮件 | P1 |
| 用户登录 | 支持用户名/邮箱登录；连续5次失败锁定15分钟 | P0 |

### 1.4 API 请求处理

| 功能点 | 验收标准 | 优先级 |
|-------|---------|--------|
| 请求认证 | 无效令牌返回401；缺少令牌返回401 | P0 |
| 请求路由 | 根据模型名称匹配渠道；无匹配渠道返回404 | P0 |
| 流式响应 | 支持SSE流式输出；流式中断自动重连 | P0 |

| 并发限制 | 超过并发限制返回429；返回Retry-After头 | P1 |

## 二、业务规则定义

### 2.1 渠道调度规则

**调度优先级：**
1. 匹配模型名称
2. 渠道优先级（数字越大越优先）
3. 渠道权重（同优先级时按权重随机）
4. 渠道状态（故障渠道不参与调度）

**故障判定标准：**

| 条件 | 判定结果 |
|------|---------|
| 连续3次请求超时 | 暂时不可用（5分钟后重试） |
| 连续5次请求失败 | 自动禁用（需手动启用） |
| 响应时间>10秒 | 标记为慢响应 |
| HTTP 5xx 错误 | 计入失败次数 |
| HTTP 4xx 错误 | 不计入失败（客户端错误） |

**恢复机制：**
- 暂时不可用：5分钟后自动恢复探测
- 自动禁用：需管理员手动启用
- 启用后：先进行健康检查，通过后恢复正常调度

### 2.2 配额扣减规则

**扣减时机：**
- 请求成功后扣减（非请求时）
- 流式响应在结束时一次性扣减

**扣减计算：**

```
扣减额度 = 输入Token × 输入倍率 + 输出Token × 输出倍率
```

**默认倍率：**

| 模型类型 | 输入倍率 | 输出倍率 |
|---------|---------|---------|
| GPT-4 系列 | 1.0 | 2.0 |
| GPT-3.5 系列 | 1.0 | 1.0 |
| Claude 系列 | 1.0 | 3.0 |
| 国产模型 | 1.0 | 1.0 |
| Embedding | 1.0 | 0 |

**配额不足处理：**
- 请求前检查：配额不足直接拒绝
- 请求后检查：扣减后为负，记录欠费，禁止后续请求

**配额重置：**
- 按月重置：每月1日0点重置为分配额度
- 不重置：累计使用，需手动充值

### 2.3 令牌权限规则

**接口权限矩阵：**

| 接口 | 普通令牌 | 管理员令牌 |
|------|:-------:|:---------:|
| /v1/chat/completions | ✅ | ✅ |
| /v1/completions | ✅ | ✅ |
| /v1/embeddings | ✅ | ✅ |
| /v1/images/generations | 可配置 | ✅ |
| /v1/audio/* | 可配置 | ✅ |
| /v1/models | ✅ | ✅ |
| 管理接口 | ❌ | ✅ |

**IP白名单规则：**
- 未配置：不限制IP
- 已配置：仅允许列表内IP访问
- 支持格式：`192.168.1.1`、`192.168.1.*`、`192.168.1.0/24`

**域名限制规则：**
- 仅对前端调用生效
- 检查请求头 `Origin` 或 `Referer`
- 支持通配符：`*.example.com`

### 2.4 并发与限流规则

**并发限制：**

| 层级 | 默认限制 | 说明 |
|------|---------|------|
| 系统级 | 1000 QPS | 整体请求上限 |
| 用户级 | 100 QPS | 单用户请求上限 |
| 令牌级 | 可配置 | 单令牌请求上限 |
| 渠道级 | 可配置 | 单渠道请求上限 |

**限流策略：**
- 超限返回 HTTP 429
- 响应头包含 `X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`
- 响应体包含 `retry_after` 字段（秒）

### 2.5 计费规则

**计费模式：**

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 预付费 | 先充值后使用，余额不足停服 | 个人用户 |
| 后付费 | 月底结算，超额部分按量计费 | 企业用户 |
| 套餐制 | 固定月费，含定量额度，超额另计 | SaaS 服务 |

**充值规则：**
- 最低充值：10元
- 充值到账：实时
- 发票：支持申请电子发票

**退款规则：**
- 未使用余额可申请退款
- 退款收取5%手续费
- 退款周期：3-5个工作日

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| API 响应时间 | P95 < 100ms（网关层） | APM 监控 |
| 吞吐量 | ≥ 1000 QPS | 压力测试 |
| 并发连接 | ≥ 10000 | 压力测试 |
| 流式首字延迟 | P95 < 500ms | 端到端测量 |
| 页面加载时间 | < 2s | Lighthouse |
| 列表加载时间 | < 500ms | 前端监控 |

### 4.2 可用性要求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | ≥ 99.9%（年停机 < 8.76h） |
| 计划内维护 | 每月1次，凌晨2-4点 |
| 故障恢复时间 | < 15分钟 |
| 数据备份 | 每日备份，保留30天 |
| 灾难恢复 | RTO < 4h，RPO < 1h |

### 4.3 安全要求

| 要求 | 说明 |
|------|------|
| 数据传输 | 全站 HTTPS，TLS 1.2+ |
| 数据存储 | 敏感数据加密存储（AES-256） |
| 密码策略 | 最少8位，含大小写+数字 |
| 会话安全 | JWT 有效期 24h，支持吊销 |
| 日志脱敏 | 日志中不记录完整 Key、密码 |
| SQL 注入 | 使用参数化查询 |
| XSS 防护 | 输入过滤 + 输出编码 |
| CSRF 防护 | Token 验证 |

### 4.4 兼容性要求

**浏览器支持：**

| 浏览器 | 最低版本 |
|--------|---------|
| Chrome | 90+ |
| Firefox | 88+ |
| Safari | 14+ |
| Edge | 90+ |

**移动端支持：**
- 响应式设计，支持 375px - 1920px 宽度
- 核心功能在移动端可用
- iOS Safari / Android Chrome 支持

**API 兼容：**
- OpenAI API 格式兼容
- 向后兼容：v1 API 至少支持 12 个月

### 4.5 可维护性要求

| 要求 | 说明 |
|------|------|
| 代码规范 | 遵循语言官方代码规范 |
| 文档覆盖 | API 文档 100% 覆盖 |
| 日志规范 | 统一日志格式，含请求ID |
| 监控告警 | 核心指标 100% 覆盖 |
| 部署方式 | 支持蓝绿部署 / 滚动更新 |

## 五、MVP 定义与迭代计划

### 5.1 MVP 范围（P0）

**目标：** 4周内交付可用的最小产品

| 模块 | 功能 | 说明 |
|------|------|------|
| 渠道管理 | 新增/编辑/删除/测试 | 仅支持 OpenAI 格式渠道 |
| 令牌管理 | 生成/查看/删除 | 支持配额限制 |
| API 代理 | 请求转发 | 仅 /chat/completions |
| 基础统计 | 请求数/Token数 | 按日统计 |
| 用户系统 | 注册/登录 | 仅邮箱登录 |

**MVP 不包含：**
- 多渠道负载均衡
- 高级计费
- 移动端适配
- 第三方登录

### 5.2 迭代计划

**V1.0（MVP + P1）— 8周**

| 迭代 | 周期 | 内容 |
|------|------|------|
| Sprint 1-2 | W1-W4 | MVP 功能 |
| Sprint 3-4 | W5-W6 | 负载均衡、多供应商支持 |
| Sprint 5-6 | W7-W8 | 用户管理、日志中心、优化 |

**V1.1 — +4周**

| 内容 |
|------|
| 充值中心 |
| 高级统计分析 |
| 移动端适配 |
| 性能优化 |

**V1.2 — +4周**

| 内容 |
|------|
| 语义缓存 |
| Prompt 模板 |
| 多语言 UI |
| API 开放平台 |

### 5.3 优先级矩阵

```
                    业务价值
              低           高
        ┌─────────┬─────────┐
   低   │  P2     │   P1    │
实现    │ 后续迭代 │ 第一版  │
成本    ├─────────┼─────────┤
   高   │  不做   │   P0    │
        │  或外包  │  MVP   │
        └─────────┴─────────┘
```

**P0（MVP 必须）：**
- 渠道基础管理
- 令牌基础管理
- API 代理
- 基础统计
- 用户登录

**P1（第一版）：**
- 负载均衡
- 多供应商
- 用户管理
- 日志中心
- 配额预警

**P2（后续迭代）：**
- 充值中心

## 六、用户流程图

### 6.1 首次使用流程

```
注册账号 → 验证邮箱 → 登录系统 → 添加渠道 → 测试连通性 → 创建令牌 → 开始调用
```

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  注册账号  │───→│  验证邮箱  │───→│  登录系统  │───→│  添加渠道  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                    │
┌──────────┐    ┌──────────┐    ┌──────────┐        │
│  开始调用  │←───│  复制Key  │←───│  创建令牌  │←───────┘
└──────────┘    └──────────┘    └──────────┘
```

### 6.2 日常使用流程

```
登录 → 查看仪表盘 → 检查用量/告警 → 调整配置（如需）→ 继续使用
```

```
收到告警 → 查看日志 → 定位问题 → 处理（禁用/切换/联系供应商）→ 验证恢复
```

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  收到告警  │───→│  查看日志  │───→│  定位问题  │
└──────────┘    └──────────┘    └──────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
```

## 七、异常流程处理

### 7.1 认证异常

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 缺少令牌 | 401 | Missing API key | 检查请求头是否包含 Authorization |
| 令牌无效 | 401 | Invalid API key | 检查令牌是否正确 |
| 令牌已过期 | 401 | Token expired | 联系管理员续期或创建新令牌 |
| 令牌已吊销 | 401 | Token revoked | 创建新令牌 |
| IP 不在白名单 | 403 | IP not allowed | 联系管理员添加 IP |

### 7.2 配额异常

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 配额不足 | 402 | Quota exceeded | 充值或申请增加配额 |
| 用户已禁用 | 403 | User disabled | 联系管理员 |
| 模型不可用 | 403 | Model not available | 检查令牌权限设置 |

### 7.3 请求异常

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 模型不存在 | 404 | Model not found | 检查模型名称是否正确 |
| 无可用渠道 | 503 | No available channel | 联系管理员添加渠道 |
| 上游超时 | 504 | Upstream timeout | 稍后重试 |
| 请求过于频繁 | 429 | Rate limit exceeded | 等待 retry_after 秒后重试 |

### 7.4 系统异常

| 场景 | 错误码 | 错误信息 | 处理建议 |
|------|--------|---------|---------|
| 内部错误 | 500 | Internal server error | 联系技术支持 |
| 服务维护 | 503 | Service maintenance | 等待维护结束 |
